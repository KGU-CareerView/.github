## Rules

저희 팀은 원활한 코드 리뷰와 히스토리 파악을 위해 아래 규칙을 따릅니다.

#1. 커밋 메시지 구조
`type: 주제 (제목)` 형식으로 작성합니다.

#2. Type 종류
- feat: 새로운 기능 추가
- fix: 버그 수정
- docs: 문서 수정 (README, 계획서 등)
- style: 코드 포맷팅, 세미콜론 누락 등 (코드 변경 없음)
- refactor: 코드 리팩토링
- CI/CD: devOps 배포 관련
- chore: 빌드 업무, 패키지 매니저 설정 등 (기타 변경)

#3. 예시
- `feat: MediaPipe 안면 랜드마크 추출 모듈 추가`
- `fix: 영상 스트리밍 레이턴시 문제 해결`
- `docs: 프로젝트 수행계획서 업데이트`
